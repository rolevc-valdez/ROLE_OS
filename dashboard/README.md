# ROLE OS Dashboard

Read-only FastAPI service over the SQLite knowledge base produced by the
ROLE OS Builder (`/builder`). No AI features are implemented yet — this is a
plain data-access API.

## Endpoints

| Method | Path                 | Description                                   |
|--------|----------------------|------------------------------------------------|
| GET    | `/health`            | Service and database connectivity status        |
| GET    | `/projects`          | Projects with conversation counts               |
| GET    | `/search?q=`         | Search knowledge cards by title/summary/content |
| GET    | `/knowledge/{id}`    | Full knowledge card by `conversation_id`        |

Interactive docs are available at `/docs` once the app is running.

## Setup

```bash
cd dashboard
python -m venv .venv
source .venv/bin/activate  # .venv\Scripts\activate on Windows
pip install -r requirements.txt
```

## Configuration

The app reads the SQLite database path from the `ROLE_OS_DB_PATH` environment
variable. It defaults to `samples/role_os_sample/00_SYSTEM/role_os.db`
(relative to the working directory) for local smoke-testing against the
bundled sample data.

To point the dashboard at a real ROLE Knowledge OS generated by the builder:

```bash
export ROLE_OS_DB_PATH="/path/to/ROLE_KNOWLEDGE_OS/00_SYSTEM/role_os.db"
```

## Run

```bash
uvicorn app.main:app --reload
```

Then visit `http://127.0.0.1:8000/health`.
